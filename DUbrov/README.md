git status - является ли папка репозиторием, если да-какие последние изменения были

git init команда иницилизирует данную папку как гид репозиторий(недобавляем гитрепозиторий в середину гитрепозетория)
git add .(название файла) - выбирает папки и файлы которые должны были отправить в комит.Точка обозначает -все файлы и папки.
git commit -m "first commit" - добавляет в буфер . обязательно флаг -m 'обязательный комментарий' 
git branch -M main -переименовать ветку main - названине ветки на которую нужно переименовать 
git remote add origin https://github.com/tepliydom91-lang/git.git привязка локального репозитория к удаленному альтернативная команда клонирует удаленный репозиторий локально 
git clone https://github.com/tepliydom91-lang/git.git клонирует отдаленный репозиторий локально 
git push -u origin main отправка в отдаленный репозиторий 
git push - отправка в удаленный репозиторий последующие разы после команды 9
git config --global user.email "you@example.com" привязка почты локально 
git config --global user.name "Your Name" привязка имени локально 
git branch - посмотреть перечень веток и в какой я нахожусь 
git branch -d/-D - удаление ветки глобально - локально / локально 
git checkout name - переход на ветку с названием 
git checkout -b name - создание ветки и переход на нее 
git rebase - при возникновении неразрешенных конфликтов в отдаленной ветке и локальной ветки 
git stash - отложить код на время . код не уберется весь , а только ново-добавленный 
git stash pop - вернуть код , который убрал 
git stash git pull - затянуть последние изменения
